# 🧠 Magic Brain Architecture: v1.0-agent-loop

## 🚀 Обзор
Масштабируемая архитектура агента, основанная на **декларативных пачках** и **иерархическом SQLite-роутинге**. LLM никогда не видит полный список всех инструментов, выбирая их из узких категорий.

## 🏗️ Компоненты

### 1. 🗄️ SQLite Tool DB (`agents/brain/tool_db.py`)
Единый источник истины о доступных инструментах.
- **Таблица `tools`**: Хранит `name`, `domain`, `category`, `subcategory`, `params_json` и путь к функции `func_path`.
- **Таблица `categories`**: Содержит человекочитаемые описания категорий для LLM.
- **Роутинг**: `get_routes()` делает JOIN с описаниями, позволяя LLM выбирать контекст (`local/system/shell` vs `local/filesystem/files`).

### 2. 📦 Declarative Packs (`agents/tools/packs/*.py`)
Инструменты не регистрируются вручную. Каждый пак — это Python-файл со списком `__skills__`.
- **Контракт**: Каждый скилл обязан объявить `category` и `name`.
- **Валидация**: Сеидер проверяет наличие обязательных полей перед записью в БД.
- **Приоритет**: Локальные паки имеют приоритет над MCP-серверами (защита от коллизий имен).

### 3. 🌱 Seeder (`agents/brain/tool_seeder.py`)
Утилита для синхронизации кода и БД.
- Сканирует `agents/tools/packs/*.py`.
- Валидирует схему (`REQUIRED_FIELDS`).
- Регистрирует валидные инструменты и категории.
- **Без хардкода**: Никаких `if "read" in name`. Данные берутся только из декларации пака.

### 4. 🧠 Planner (`agents/brain/planner.py`)
Двухэтапный планировщик (LLM + SQLite):
1. **Intent Routing**: LLM выбирает **одну** ветку маршрута (например, `local/memory/kv`) из списка доступных категорий.
2. **Tool Planning**: LLM получает только инструменты выбранной ветки и строит JSON-план.
- **Анти-галлюцинация**: Температура `0.0`. Пост-валидация: если LLM вернула имя инструмента, которого нет в ветке — шаг отбрасывается.

### 5. ♻️ Agent Loop (`agents/brain/agent_loop.py`)
Оркестратор выполнения:
- **Выполнение**: Передача аргументов через `**kwargs` (поддержка и нативных функций, и MCP-обёрток).
- **Resilience**: При ошибке выполняет `replan()` (макс. 2 попытки).
- **Fallback**: Если план пустой или все попытки исчерпаны — переходит в обычный чат-режим.

## 📊 Схема данных (Tools Table)
| Column | Type | Описание |
|--------|------|----------|
| `name` | TEXT PK | Уникальное имя инструмента |
| `domain` | TEXT | Глобальная зона (local, cloud) |
| `category` | TEXT | Группа (filesystem, system, memory) |
| `subcategory`| TEXT | Подгруппа (files, shell, kv) |
| `params_json`| TEXT | JSON-схема аргументов функции |
| `func_path` | TEXT | Путь `module:function` для импорта |

## 🔐 Безопасность
- **Приватность**: Разделение на `CLOUD` и `LOCAL` (реализовано через теги).
- **Валидация**: Строгая проверка типов аргументов при исполнении.
